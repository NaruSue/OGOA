# 下位モデル向け実装ガイド

このガイドは、1G1A の次回実装で迷いやすい部分をまとめたものです。実装はこの順番で進めると安全です。

## 1. 文字化けの解消

最優先です。既存の PHP と docs の一部に文字化けした日本語が残っている場合があります。

- HTML の `Content-Type` は `text/html; charset=UTF-8`
- `<meta charset="utf-8">` を出力する
- PHP ファイルは UTF-8 で保存する
- 画面文言は日本語として読み直せる状態にする

## 2. DB を共有イベント中心へ寄せる

推奨テーブル:

- `share_events`
- `share_event_photos`
- `share_access_logs`

既存の `posts` を使い続ける場合でも、意味は「QR表示1回の共有イベント」として扱います。

必要な保存項目:

- 選択された共有プロフィール
- 今回だけのコメント
- 今回だけの写真
- QR表示時の位置情報
- QR表示日時

## 3. 管理トップをモックに合わせる

`/dashboard` はスマホ利用を優先します。

- 上部に共有プロフィール選択
- コメント入力
- 写真追加
- `QRを公開` ボタン
- 下部に共有プロフィール一覧
- `+` で共有プロフィール追加

`QRを公開` を押したときは、共有イベントを作成して QR 画面へ遷移します。

初回ログイン時は、空の管理画面を見せずに、説明文つきの初期共有プロフィールを1件自動作成してください。ユーザーには「このサービスは会った相手に見せたい共有プロフィールを QR で渡すためのもの」という説明をログイン直後に見せると分かりやすいです。

通信できない場合でも QR 表示を優先します。

- 端末に予約済み `public_token` があれば、それを使ってローカル下書きを作る
- コメント、写真、位置情報は IndexedDB に保存する
- QR 画面は `/s/{public_token}` を表示する
- 通信復帰時にローカル下書きをサーバへ同期する
- 同期状態は `未同期`, `同期中`, `同期済み`, `失敗` が分かるようにする

## 3.5. 予約トークンと同期キュー

オンライン時に、共有イベント用トークンを事前取得して端末に保存します。

実装するもの:

- 予約トークン取得エンドポイント
- IndexedDB の `reservedTokens`
- IndexedDB の `pendingShareEvents`
- 未同期データを送る同期処理
- 同期失敗時の再試行

最低限の同期タイミング:

- 管理トップ表示時
- QR 表示画面表示時
- ブラウザの `online` イベント発火時

注意:

- 写真は localStorage に入れない
- 写真 Blob は IndexedDB に保存する
- 同期APIは `public_token` をキーにして冪等にする
- 同期前にゲストがアクセスした場合は準備中画面を出す

## 4. QR 表示画面

QR は共有イベントURLを指すようにします。

推奨URL:

```text
/s/{share_event_token}
```

QR画面で表示するもの:

- 大きな QR
- 短いURL
- 日本語メッセージ
- 英語メッセージ

位置情報:

- QR を公開するときに Geolocation API を呼ぶ
- 許可されたら緯度、経度、精度を保存する
- 拒否されたら何もしない
- QR 表示は止めない

## 5. ゲスト初回名前入力

`/s/{share_event_token}` で Cookie を確認します。

Cookie がない場合:

- 名前またはニックネーム入力画面を表示する
- POST 後に `guest_visitors` へ保存する
- `1g1a_guest_token` と `1g1a_guest_name` を Cookie に保存する
- 同じ共有イベント画面へリダイレクトする

Cookie がある場合:

- 名前入力画面を省略する
- 共有プロフィール画面を表示する

## 6. ゲスト公開プロフィール画面

表示内容:

- `○○さん、ようこそ`
- 共有プロフィールの表示名
- プロフィール写真（未登録時は公開表示名から生成した代替表示）
- 見出し
- 紹介文
- `今日のメッセージ`
- `今日の写真`
- `SNS・リンク`

表示しないもの:

- 編集ボタン
- 管理画面リンク
- QR
- プロフィール選択
- ログイン導線
- 位置情報

## 6.5. 共有プロフィールの SNS リンク編集

共有プロフィールの編集画面で、SNS や Web サイトのリンクを入力できます。

- SNS 種類を選ぶ
- 表示名を入れる
- URL を入れる
- 必要なら並び順を指定する
- 保存すると、その共有プロフィールの公開ページとゲスト画面に表示される
