# 1G1A 要件

## 目的

1G1A は、初対面の相手に「連絡先を直接渡す」のではなく、「その場で見せる共有ページ」を渡すためのサービスです。

相手は登録やログインをせずに QR コードを開けます。後からページを見て、必要だと思ったときだけ SNS につなげます。

## MVP でやること

- Google ログイン
- ユーザー登録とプロフィール管理
- 1 ユーザーが複数プロフィールを持てる
- プロフィールごとの SNS 管理
- 共有ページの QR コード表示
- 共有ページの閲覧ログ

## MVP ではやらないこと

- 不特定多数への完全公開
- 位置情報の常時収集
- 画像の高度な加工
- SNS の自動投稿
- NoSQL の採用

## 重要な考え方

- プロフィール単位で公開/非公開を切り替える
- SNS は JSON ではなく別テーブルで管理する
- 個人情報は最小限にする
- 誰がどこまで見られるかを明確にする

## 主要ユースケース

1. Google でログインする
2. プロフィールを作成する
3. SNS を登録する
4. QR コードを発行する
5. 相手が QR から共有ページを見る
6. 必要なら SNS へ遷移する
