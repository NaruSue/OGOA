# 共有イベントの地図アプリリンク追加

## 追加したい機能

共有イベントに位置情報が保存されている場合、イベント画面のイベント情報から地図アプリを開けるリンクを表示する。

## 表示条件

- 共有イベントに緯度・経度が保存されていること
- 位置情報の取得に失敗している場合はリンクを表示しない
- 位置情報が未保存のイベントでは何も表示しない

## 表示場所

- イベント画面の「イベント情報」内
- 位置情報の表示行、またはその直下に「地図で開く」ボタンを置く

## 挙動

- タップすると端末の既定の地図アプリ、またはブラウザの地図ページを開く
- iPhone では Apple Maps、Android では Google Maps、PC ではブラウザの地図ページに遷移できるようにする
- 開ける形式は端末依存なので、実装では `geo:` URI か HTTPS の地図 URL を併用する

## 仕様メモ

- 共有イベントの位置情報は、QR 公開時に取得した緯度・経度を使う
- ゲスト画面には位置情報を表示しない
- 地図リンクは管理者本人向けの導線として扱う

## 実現性

実現できる。

理由:

- 緯度・経度があれば、外部地図アプリを開く URL を生成できる
- ブラウザからでも `geo:` や Google Maps / Apple Maps の URL に遷移できる
- 既存の位置情報保存設計と矛盾しない

注意点:

- `geo:` URI は端末やブラウザによって挙動が異なる
- そのため、最終的には HTTPS の地図 URL をフォールバックとして持たせるのが安全
- 位置情報の公開範囲は、管理画面内に限定する前提を維持する
